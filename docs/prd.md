## 1. 产品概述 (Product Overview)

### 1.1 产品简介

Cross Chat 是一款面向 B 站主播的 OBS 直播增强工具。它将**主播实时语音字幕**、**观众弹幕流**和**智能匹配回复高亮**融合为单一可视化叠加层，直接叠加到直播画面上，提升观众互动体验和直播内容可消费性。

### 1.2 核心问题

| 问题 | 现状 | 目标 |
|------|------|------|
| 主播说了什么？ | 新观众进入直播间不知主播在聊什么 | 实时字幕让观众 1 秒理解当前话题 |
| 主播在回复谁？ | 弹幕滚得快，主播口头回复看不清对应关系 | 箭头连线 + 高亮显示"被回复的弹幕" |
| 弹幕杂乱无章 | 传统弹幕从右飘过，速度和密度不可控 | 线性垂直滚动栏，清晰可读 |

### 1.3 目标用户

- **主要用户**: B 站游戏/聊天类主播，希望通过更好的互动视觉效果提升直播间观看体验
- **次要用户**: 观众（受益于更清晰的内容呈现）

### 1.4 产品愿景

> 主播说一句话，弹幕自动显示"谁被回复了"——让直播间互动一目了然。

MVP 阶段先验证核心链路（语音字幕 + 弹幕 + 匹配高亮），后续迭代做语义理解和多平台支持。

## 2. 用户故事 (User Stories)

| 编号 | 角色 | 需求 | 验收条件 |
|------|------|------|----------|
| US-01 | 主播 | 我说的话能实时显示为字幕 | 主播说话后 500ms 内，画面底部出现对应字幕文字 |
| US-02 | 主播 | 观众的弹幕按时间线垂直排列并滚动 | 弹幕以列表形式从下往上滚动，新弹幕出现在底部 |
| US-03 | 主播 | 我回复某条弹幕时，该弹幕自动高亮 | 主播说的话与某条弹幕关键词匹配时，弹幕高亮并显示箭头连线 |
| US-04 | 观众 | 一眼看出主播在回复谁的问题 | 被回复的弹幕有明显的视觉标识（颜色高亮 + 箭头） |

## 3. 功能需求 (Functional Requirements)

### 3.1 语音转字幕模块 (FR-VOICE)

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| FR-V01 | 实时采集麦克风音频并转为中文文本 | P0 | 使用 obs-localvocal (Whisper.cpp) 本地推理 |
| FR-V02 | 字幕文本输出到 OBS 可读的 Text Source | P0 | 支持 GDI+ 文本源，主播可在 OBS 中自由调整位置和样式 |
| FR-V03 | 字幕文本同时输出到本地文件 | P1 | 供 Python 后端读取，做匹配分析 |

### 3.2 弹幕采集模块 (FR-DANMU)

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| FR-D01 | 接入 B 站直播间弹幕 WebSocket 流 | P0 | 通过直播间 ID 连接，无需登录 |
| FR-D02 | 实时接收并缓存弹幕消息 | P0 | 包含用户昵称、弹幕内容、时间戳 |
| FR-D03 | 维护滑动窗口（最近 30 条弹幕） | P0 | FIFO 策略，超过 30 条淘汰最旧的 |
| FR-D04 | 断线自动重连 | P1 | 指数退避，最大间隔 30s |

### 3.3 匹配引擎模块 (FR-MATCH)

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| FR-M01 | 将主播最新一句话与最近弹幕做匹配 | P0 | MVP: 基于关键词交集；后续: embedding 语义匹配 |
| FR-M02 | 匹配命中时标记弹幕 ID | P0 | 返回匹配的弹幕 ID 列表及关键词 |
| FR-M03 | 匹配超时自动取消 | P1 | 高亮持续 3 秒后自动消失 |

### 3.4 可视化 UI 模块 (FR-UI)

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| FR-U01 | 线性垂直滚动弹幕列表 | P0 | 新弹幕从底部出现，旧弹幕向上滚动消失 |
| FR-U02 | 主播字幕条（底部/顶部固定） | P0 | 显示主播最新一句完整文本 |
| FR-U03 | 匹配弹幕高亮效果 | P0 | 金色背景闪烁 + 箭头连线指向字幕条 |
| FR-U04 | 弹幕列表最多显示 30 条 | P0 | 超出部分自动移除 DOM 节点，控制内存 |
| FR-U05 | UI 叠加层透明背景 | P0 | 不影响直播画面，通过 CSS `background: transparent` |
| FR-U06 | 弹幕项包含用户昵称和内容 | P0 | 显示格式: `[用户昵称]: 弹幕内容` |

### 3.5 通信模块 (FR-COMM)

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| FR-C01 | 后端通过 WebSocket 推送状态到前端 | P0 | `ws://localhost:8765`，JSON 格式 |
| FR-C02 | 前端页面加载后自动连接 WebSocket | P0 | 连接失败时显示状态提示，自动重连 |
| FR-C03 | 支持增量推送和全量快照两种模式 | P1 | 低负载用增量事件，首次连接用全量快照 |

## 4. 非功能需求 (Non-Functional Requirements)

### 4.1 性能

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 语音→字幕延迟 | < 500ms | 从麦克风输入到 OBS Text Source 更新 |
| 弹幕→显示延迟 | < 200ms | 从 B 站推送到 Browser Source 渲染完成 |
| 匹配计算延迟 | < 50ms | 关键词匹配函数耗时 |
| Browser Source 帧率 | ≥ 30fps | Chrome DevTools Performance |
| CPU 占用 (Python 进程) | < 5% | 任务管理器 |
| 内存占用 (Browser Source) | < 100MB | Chrome DevTools Memory |

### 4.2 可靠性

| 指标 | 目标值 |
|------|--------|
| 弹幕连接稳定性 | 99% 在线率（1 小时直播中断 <36 秒） |
| WebSocket 连接恢复 | 断线后 5 秒内自动重连 |
| 语音识别可用性 | 直播全程持续输出字幕（不因 OBS 操作中断） |

### 4.3 兼容性

| 组件 | 要求 |
|------|------|
| OBS Studio | 30.0+ (Windows/macOS/Linux) |
| Python | 3.11+ |
| 浏览器引擎 | Chromium Embedded (OBS 内嵌, ≥96) |
| 网络 | 能访问 B 站弹幕 WebSocket 服务 |

## 5. MVP 范围 (MVP Scope)

### 5.1 包含 (In Scope)

- [x] 主播实时语音转字幕（obs-localvocal 集成）
- [x] B 站弹幕实时捕获和垂直列表显示
- [x] 基于关键词的简单匹配规则
- [x] 匹配高亮 + 箭头连线动画
- [x] OBS Browser Source 可视化叠加
- [x] Python WebSocket 本地通信桥梁

- [ ] 多平台弹幕支持（抖音、Twitch、YouTube）

- [ ] 反垃圾/弹幕过滤
- [ ] 自定义主题和样式面板
- [ ] LLM 上下文理解
- [ ] 多人语音场景（连麦）
- [ ] 礼物/SC 等其他直播间事件

## 7. 技术约束 (Technical Constraints)

| 约束 | 说明 |
|------|------|
| 必须运行在主播本机 | 所有服务本地运行，不依赖云端（除 B 站弹幕 API） |
| 必须兼容 OBS Browser Source | 前端代码只能使用 Chromium Embedded 支持的特性 |
| 语音识别必须本地 | 不能使用云端 ASR（隐私 + 延迟） |
| 语言 | 中文为主，英语可后续扩展 |

## 8. 风险与缓解 (Risk Assessment)

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| obs-localvocal 模型下载慢/安装失败 | 阻断 MVP | 中 | 提供手动下载教程 + 备用 faster-whisper Python 方案 |
| blivedm 库 B 站协议变更导致不可用 | 阻断弹幕 | 中 | 关注库更新；备选方案：直接逆向 B 站 WS 协议 |
| WebSocket 本地端口冲突 | 页面无法连接 | 低 | 支持配置端口号 |
| Browser Source 内存泄漏 | 长期直播卡顿 | 中 | 限制 DOM 节点数、定期清理、60fps 性能预算 |
| Whisper 模型精度不足 | 识别错误导致匹配质量差 | 中 | 使用 larger 模型（需权衡延迟）；用户可手动切换模型 |

## 9. 附录 A: 快速开始指南

### 步骤 1: 安装语音转字幕 (5 分钟)

1. 下载 [obs-localvocal](https://github.com/royshil/obs-localvocal/releases) 最新版
2. OBS → 麦克风音频源 → 右键 → 滤镜 → 添加 **LocalVocal**
3. 确认 OBS 中出现 Text (GDI+) 字幕源
4. （可选）开启"输出到文件"功能

### 步骤 2: 安装 Python 环境

```bash
pip install blivedm websockets
```

### 步骤 3: 启动后端

```bash
cd backend/
# 编辑 mvp_backend.py 中的 ROOM_ID 为你的直播间 ID
python mvp_backend.py
```

### 步骤 4: 配置 OBS Browser Source

1. OBS → 添加源 → **Browser Source**
2. URL 填入: `file:///path/to/cross_chat/frontend/index.html`
3. 调整位置、尺寸、透明度

### 步骤 5: 开播测试

开始直播 → 说话 → 观察字幕和弹幕匹配效果。
